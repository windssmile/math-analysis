## 1. 项目目标

本项目拟编写一部面向普通高校数学类专业本科生的中文《数学分析》数字教材。教材完整覆盖传统数学分析主干内容，同时把数值计算与数学建模发展为可独立选学、又与理论互相支撑的第二条主线。

教材以自主学习为首要场景。读者应能借助明确的先备知识、完整的核心证明、分阶段撤除的学习脚手架、即时反馈、习题答案和可复现的 Python 实验，在缺少教师持续讲解的情况下推进学习。

全书不追求通过增加专题来制造“内容丰富”的表象。所有内容必须服务于以下终点：读者能够理解数学分析的核心结构，独立阅读和书写证明，正确选择解析方法，把适当的理论转化为数值算法，并能够建立和批判数学模型。

## 2. 依据与课程边界

国内数学类专业通常把数学分析设置为本科低年级的多学期专业基础课。复旦大学公开大纲采用三学期、272 学时结构，内容从实数与极限延伸到多元分析、重积分和向量分析；其他高校公开大纲常见总量约 240–324 学时。本书据此保留经典数学分析体系，并为应用、算法、建模、Fourier 级数和实分析入口留出正式空间。原定 360 等效学时是课程规划基线，不是压缩自学论证和训练的硬上限；各部完成自学教材级蓝图后按真实学习负担重新核算。

参考入口：

- [复旦大学《数学分析》教学大纲](https://math.fudan.edu.cn/anal/34382/list.htm)
- [重庆师范大学《数学分析 I–III》课程大纲](https://math.cqnu.edu.cn/info/1037/6997.htm)
- [长江大学数学与应用数学专业课程教学大纲](https://shuxue.yangtzeu.edu.cn/__local/1/35/A5/B05973F0D0CE2CDFBD1F26162F2_8F5E7DB9_3F6D6D.pdf)

### 2.1 纳入理论主线

- 实数完备性、数列极限、函数极限与连续性；
- 一元微分学、微分中值定理、Taylor 公式；
- 不定积分、Riemann 积分、反常积分；
- 数项级数、函数列与函数项级数、一致收敛、幂级数；
- Euclid 空间中的极限与连续、多元微分、隐函数与极值；
- 重积分、曲线积分、曲面积分、Green/Gauss/Stokes 公式；
- 含参变量积分；
- 测度、可测函数、Lebesgue 积分及三大收敛定理的正式入门；
- Fourier 级数、正交体系、Bessel 不等式、Parseval 恒等式及均方收敛。

### 2.2 纳入应用选修线

- 数值求根、迭代与停止准则；
- 插值、逼近与误差分析；
- 数值微分与数值积分；
- 一元及多元优化；
- 几何、物理、质量、流量、概率和周期现象中的模型；
- 参数敏感性与综合建模项目；
- Python 实现、图形展示与结果解释。

### 2.3 明确不扩张的范围

- 不系统教授完整实变函数课程中的乘积测度、Radon–Nikodym 定理和一般 \(L^p\) 理论；
- 不系统教授 Fourier 变换和偏微分方程；
- 不把应用线写成压缩版《数值分析》或《数学建模》；
- 不从自然数开始完整构造实数。正文采用有序域与完备性公理，实数构造放入附录；
- 不单设脱离数学内容的“证明预备课”。证明语言和策略在前期真实内容中逐步引入；
- 不重讲高等代数、解析几何和 Python，只提供当前单元所需的精准知识桥。

## 3. 核心设计原则

### 3.1 依赖骨架与问题弧线的双螺旋

宏观顺序遵守严格的数学依赖。每一部同时设置一个持续数章的核心问题：理论线通过定义、定理和证明建立工具，应用线把同一工具转化为算法或模型，两条线最终在该部的汇合单元或综合项目中共同回答开篇问题。

这种结构不要求每章机械配置相同数量的理论与应用内容。内容形态由学习目标决定，统一性来自问题弧、依赖关系和能力进阶，而不是栏目数量相同。

### 3.2 问题驱动，但不牺牲严格性

核心主题尽量经历“问题—现象—猜想—定义—证明—算法—模型”的发展过程。七个阶段不必在每个小节全部出现，但定义和定理必须回应已经暴露的困难，算法与模型必须说明依赖的理论条件。

### 3.3 核心证明完整

所有核心定理和关键推论给出完整证明。正文不得用“显然”“容易验证”等措辞掩盖决定结论成立的步骤。核心证明按“障碍—路线—逐步证明—假设使用位置—易错边界—方法迁移”组织，使读者既能看到策略，也能检查量词、指标和逻辑。重复性强的技术引理可设计为引导式习题；非核心且篇幅很长的证明可收入附录，但正文必须明确依赖和入口。

### 3.4 算法先于代码

所有计算主题遵循同一顺序：

> 问题来源 → 数学转化 → 算法思想 → 误差与适用条件 → 伪代码 → Python 实现 → 结果解释

Python 使用 NumPy、SciPy、SymPy 和 Matplotlib 等常见生态，但正文必须做到不运行代码也能理解数学与算法。代码不能代替推导。

短实验可以进入理论单元，用来形成猜想、检查边界或暴露反例；完整算法集中在应用单元。教材中的核心算法必须在 `src/` 中保留唯一实现并接受自动化测试，正文不得维护一份与测试实现脱节的复制品。

## 4. 学时与内容层级

全书当前按约 384 个等效学时规划，其中理论主线约 290 学时，应用选修线约 94 学时。一个等效学时按 45–50 分钟估算；自主学习者的实际用时可因练习量和基础不同而变化。该总量是在第二、三部完成自学教材级重估后的阶段性基线，后续各部不得为了维持旧的 360 学时数字而删减必要证明、例题或训练；待十二部蓝图完成后统一复核总量与理论／应用比例。

1. 12 个“部”：展示宏观知识版图和核心问题弧；

## 6. 学习单元标准

### 6.1 单元元数据

每个学习单元必须声明：

- 稳定单元 ID、所在部和章；
- 理论主线或应用选修标识；
- 建议用时与难度；
- 书内前置单元；
- 所需高等代数、解析几何和 Python 知识；
- 对应知识桥；
- 可检验的学习目标；
- 对应能力线；
- 后续去向。

外部先备知识不得只写“需要高等代数基础”。应具体到“线性映射的矩阵表示”“内积与正交”“平面曲线的参数方程”等可定位内容。

### 6.2 混合式固定闭环

每个单元用问题驱动进入，用传统的定义—定理—证明结构承载理论核心，再回到应用与检验：

1. **单元信息与先备依赖**：给出可达链接和当前真正使用的先备事实；
2. **学习目标与牵引问题**：说明本单元要解决的数学困难；
3. **探索、猜想或失败尝试**：用例子、图形、短实验或反例暴露定义需要；
4. **概念与理论**：按定义—定理—证明组织，核心证明执行 3.3 节模板；
5. **例题与迁移**：至少 2 个完整例题，形成从直接应用到方法选择的梯度；
6. **即时检验**：至少 2 个带答案的短任务；
7. **应用或实验**：按目标配置，不能代替理论证明；
8. **习题与完整解答**：至少 5 道分层习题，页面内折叠显示完整解答；
9. **误区、边界与后续接口**：指出假设失效时的反例和下一单元去向。

核心定理单元的习题必须覆盖证明、反例或辨错、条件变化和跨单元迁移；定义入门单元与综合应用单元可以按目标增加题量，不机械追求栏目等长。

### 6.3 按需栏目

- 证明工坊；
- 反例诊所；
- 算法实验；
- 建模现场；
- 高代、解析几何或 Python 知识桥；
- 能解释概念来源或方法失败原因的历史注脚。

这些栏目按学习目标配置，不要求每章或每单元全部出现。

### 6.4 章与部的收束

每章结束时提供概念依赖图、方法选择表、按需分层的综合习题和自测清单。每部结束时回到开篇核心问题，设置理论与应用汇合单元、可裁剪的综合项目，以及与后续课程和现实领域的去向图。

## 7. 习题与答案原则

习题不机械凑齐固定题型。每章可按目标选择基础计算、概念辨析、证明训练、反例构造、应用建模和算法实验，但每个正式学习单元遵守 6.2 节的最低数量与类型要求。

已经锁定的底线如下：

- 每道正式习题都有页面内折叠的完整解答；
- 每类代表性方法至少有完整解答示范；
- 复杂证明题可以先给分级提示，但提示不能代替完整解答；
- 完整解答必须解释方法选择与关键转折，而不只罗列计算步骤；
- 习题、答案、提示和完整解答通过稳定 ID 对应。

答案默认折叠，读者主动展开；稳定 ID 将题目与解答一一对应。教师用附加题如以后设置，必须与面向自学者的正式习题分离，不能成为缺少正文训练的理由。

## 8. 六条能力线与四阶段进阶

### 8.1 六条能力线

1. 概念与结构；
2. 证明与反例；
3. 解析计算；
4. 数值算法；
5. 数学建模；
6. 数学表达与自主学习。

### 8.2 四阶段

| 阶段 | 覆盖 | 证明发展 | 计算与应用发展 |
|---|---|---|---|
| 入门 | 第一至三部 | 识别量词、模仿范例、补全关键步骤 | 从定义验证结果，把存在性结论转为简单迭代 |
| 核心 | 第四至六部 | 选择中值定理、完备性、一致收敛等工具 | 兼顾精确方法与误差，完成一元优化、求积和逼近 |
| 推广 | 第七至十部 | 处理多变量、局部—整体和多定理组合 | 选择坐标与参数化，完成多元优化、质量、流量和参数模型 |
| 现代入口 | 第十一至十二部 | 在抽象定义下组织完整论证 | 比较不同收敛与积分理论，解释 Fourier 逼近与误差 |

前期允许标注量词结构、提供证明策略、算法模板和建模假设清单；后期逐步撤除这些脚手架，要求读者独立选择定理、停止准则、模型假设和验证方法。

## 9. 数字书稿与发布设计

### 9.1 唯一正文源

正文采用结构化 Markdown/Quarto，公式使用 LaTeX 语法。网页是第一阶段唯一正式发布物；源稿从一开始保留未来生成 PDF 与 EPUB 的能力，但在内容体系稳定前不为多格式输出牺牲网页体验。

Quarto Book 适合多章书稿，可生成带章节编号、跨章引用和全文搜索的静态 HTML，并有正式的 GitHub Pages 发布方案：

- [Quarto Books](https://quarto.org/docs/books/index.html)
- [Quarto 发布到 GitHub Pages](https://quarto.org/docs/publishing/github-pages.html)
- [GitHub Pages 自定义工作流](https://docs.github.com/en/pages/getting-started-with-github-pages/using-custom-workflows-with-github-pages)

### 9.2 内容组件

- 按部、章和学习单元拆分的正文；
- 课程清单、学时、依赖、能力映射和习题覆盖数据；
- 定义、定理、例题、习题和答案的稳定 ID；
- Python 实验、测试、示例数据和锁定的依赖环境；
- 图形、图表、术语表、符号表、参考文献和勘误记录。

### 9.3 构建与发布流

1. 修改学习单元及关联的习题、答案或代码；
2. 自动检查元数据、稳定 ID、交叉引用、先备依赖、单元 2.25 学时上限和答案覆盖；
3. 人工复核数学内容，自动运行 Python 示例和数值断言；
4. 构建公式、图表、代码输出、答案折叠、导航和搜索索引；
5. GitHub Actions 只把通过全部检查的静态站点部署到 GitHub Pages。

构建失败不得覆盖当前线上版本。GitHub Pages 站点应控制媒体体积，并遵守官方的仓库、站点体积和部署时长限制。

## 10. 质量保证与勘误

### 10.1 五类验收

1. **课程覆盖**：当前学时表、单元注册表与正文一致，每个课程目标均有对应单元和检验任务；全书蓝图完成后再冻结最终总学时；
2. **数学正确性**：术语与符号一致，核心定理证明完整，假设无遗漏，前向引用显式；
3. **自主学习可用性**：先备知识可达，每单元有目标和即时反馈，每道正式习题有答案；
4. **计算可信度**：算法先解释思想和误差，代码可重复运行，图表和数值结论可追溯；
5. **发布质量**：内部链接、公式、移动端布局、搜索、图片替代文本和完整构建通过检查。

上述编写与验收标准适用于全书十二部。已发布的第一部列入后续独立复审；第二、三部按专项重写设计执行；第四至十二部从首次蓝图开始直接执行本基线。

### 10.2 错误处理

- 每个网页提供“报告问题”和“编辑此页”入口；
- Issue 模板要求填写稳定的单元、定理或习题 ID；
- 数学错误、文字错误和代码错误分类处理；
- 发布版本具有版本号和日期；
- 重大数学修正进入公开勘误记录；
- 标题润色或目录调整不得改变稳定 ID；
- 目录变化后尽量保留旧链接或设置明确跳转。

## 11. 许可与署名

正文、原创图形和项目自有材料采用 CC BY-SA 4.0。引用、改编或使用第三方材料时，必须记录来源、作者、原许可和改编说明；不得因全书许可统一而错误覆盖第三方材料的许可要求。

代码的许可在首次公开发布前单独确定，并在仓库中明确区分内容许可与代码许可。

## 12. 书稿规划验收标准

本规划完成后，后续逐章设计必须能够回答以下问题：

- 该章属于哪一部，回应哪一条问题弧？
- 该章包含哪些学习单元，总学时如何分配？
- 每个单元依赖哪些书内知识、高等代数或解析几何知识？
- 该单元主要推进哪条能力线，完成标准是什么？
- 哪些核心定理必须完整证明？
- 是否需要证明工坊、反例、算法、模型或知识桥？
- 习题是否覆盖目标，且每题是否具有答案？
- Python 示例是否先解释算法并通过可重复性验证？
- 该章是否存在孤立内容、过早引用或与后续课程越界？

只有在上述问题均有明确答案后，一个章级规划才可进入正文写作。

## 13. 后续决策门

本规划通过书面审阅后，下一阶段只制定实施计划，不直接批量写正文。实施计划应先完成全书内容清单与依赖图，再选择第一至三部中的代表章制作样章，以验证单元长度、问题驱动方式、完整证明密度、答案呈现和 Quarto 发布链路。样章验证通过后，冻结习题与解答基线，再进入分批写作。
