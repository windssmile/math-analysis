# 第一部“实数、函数与分析语言”单元蓝图

## 目标与边界

本设计细化《数学分析：理论、算法与模型》的第一部，面向普通高校数学类专业本科生的自主学习场景。第一部回答的问题是：

> 有限符号怎样描述无限与连续？

它不是集合论或数理逻辑的独立课程，而是后续数学分析所需的语言、实数基础与无限过程预备。全书已采用的 360 学时基线保持不变；本部严格闭合为理论 20 学时、应用 4 学时，共 24 学时。

本部从已知的 \(\mathbb N,\mathbb Z,\mathbb Q\) 及其基本代数性质出发；不从 Peano 公理构造自然数。实数采用 Dedekind 分割作主构造，Cauchy 列只在第 4 章作为比较视角出现。

## 课程定位与比较结论

国内常见数学分析开篇通常采用“实数集与函数—数列极限”的节奏；国际严格分析教材则常先建立一般数学语言与实数系统，再进入极限。本设计采用后者的严格性，但用问题驱动、短学习单元和算法证书降低开篇门槛。

- 华东师大《数学分析》的开篇包含实数、数集与确界原理、函数，随后进入数列极限。
- 陈纪修、於崇华、金路《数学分析》从集合与映射进入数列极限。
- Zorich 先讲一般概念与记号、实数，再讲极限；Tao 先构造数系；Rudin 从实数系统开始，并将 Dedekind 分割作为附录。

## 依赖与内容边界

```text
集合、量词、证明、函数
          ↓
有理数缺口 → Dedekind 分割 → 完备实数
          ↓
上界、下界与确界原理
          ↓
递推、区间套、二分逼近与极限预理解
          ↓
第二部：数列极限的 epsilon-N 理论
```

1. 第 2 章不提前讨论 Cauchy 列。第 4 章在学习者已经接触数列、递推和误差语言后，才把它作为 Dedekind 分割的比较视角。
2. 第 4 章的二分法只逼近由前文构造或平方根存在性已保证的 \(\sqrt2\)。算法维持 \(a_n^2<2<b_n^2\)，以区间长度给出误差证书；不能在此处调用尚未建立的介值定理来断言任意方程有根。
3. 第 4 章可以用“任给误差容许量”的形式给出少量严格论证，但不系统展开 epsilon-N 定义、极限运算及收敛准则；这些属于第二部。

## 单元写作与验收合同

每个单元保留既定的八个固定二级栏目，并按需要启用基础计算、概念辨析、证明训练、反例构造、建模或算法实验。每个单元必须：

1. 明示高等代数、解析几何与 Python 的前置知识；无前置也须明确标注。
2. 提供一个概念辨析任务，以及一个证明或反例任务；第 4 章可将后者替换为算法正确性论证。
3. 设置 3–6 道正式习题并提供答案；代表性方法附完整解答。
4. 保持稳定的单元、定义、定理、例题与习题 ID。
5. 让章节末“回看问题”说明本章的工具如何成为下一章的前置，而不是重复摘要。

第一部完成后设置综合任务：为“用二分法求 \(\sqrt2\)”写出对象定义、存在性依据、算法、停止准则与误差证明。它不是新单元，作为四章闭环的验收任务。

## 执行顺序

1. 先登记 14 个单元元数据并建立章节骨架，验证 ID、学时、依赖和导航。
2. 完整撰写第 1 章，统一逻辑符号、证明难度、例题与习题答案的基线。
3. 依次撰写第 2、3、4 章；第 2 章以 Dedekind 分割为第一个长证明专题，第 3 章以确界原理建立全书证明基准。
4. 第 4 章加入两次轻量 Python 活动，并用综合任务验收。
5. 每章完成后运行 `make verify`，推送功能分支；合并 `main` 后由 GitHub Actions 构建和部署。

第一轮正文写作顺序固定为：

```text
u-01-01-01 → u-01-01-02 → u-01-01-03 → u-01-01-04
```

## 非目标

- 不在第一部完整讲授形式逻辑、模型论或集合论公理系统。
- 不从 Peano 公理构造自然数、整数和有理数。
- 不在第一部证明连续函数的介值定理或将二分法推广为任意连续方程的存在性算法。
- 不以程序实验替代数学证明或误差证书。
